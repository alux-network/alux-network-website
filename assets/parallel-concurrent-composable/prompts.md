# Runtime Lab Image Prompts

These prompts document the Runtime Lab visual exploration. `alux-runtime.webp` remains the live hero asset; the three mode studies (`parallel.webp`, `concurrent.webp`, and `composable.webp`) were superseded by the current native SVG diagrams and are no longer shipped. The ALUX logo is not generated inside the images; the page overlays the real `assets/alux-logo.png` where needed.

## Shared Negative Prompt

No text, no letters, no numbers, no code, no UI labels, no charts, no fake logo, no watermark, no pseudo-brand marks, no ALUX logo drawn by the model. Avoid stock-photo look, generic blockchain coin imagery, floating decorative blobs, bokeh orbs, sci-fi clutter, childish game-like style, and unreadable micro text.

## `alux-runtime.webp`

Use case: stylized-concept. Asset type: website hero illustration for ALUX Network Learn page.

Create a premium technical illustration of a decentralized concurrent runtime that unifies blockchain validator nodes, cloud services, personal devices, AI agents, capability tokens, and commit rails into one live service layer. The scene is a clean warm off-white technical studio space with subtle depth, real material surfaces, and a central abstract runtime engine made of glass, brushed metal, ceramic, and soft light. The subject is a central runtime core with several distinct lanes converging into it, showing long-running services, external events, and final settlement checkpoints without labels. Use a high-end editorial 3D technical illustration style with realistic materials, a sophisticated product-site visual feel, not cartoon and not game art. Wide 16:9 composition, central core slightly right of center, generous safe space on the left for website copy, all key elements fully visible and not cropped. Warm but restrained lighting, clear structure, quiet confidence, precise and premium. Palette: ivory, charcoal ink, muted teal, soft amber, restrained violet accents; avoid a one-color blue or purple gradient look.

## `parallel.webp`

Use case: stylized-concept. Asset type: website section illustration for mode state: Parallel.

Create a premium technical illustration that explains parallel execution through a travel request: flight, hotel, transfer, and insurance searches launch together on four independent execution lanes. The scene is a clean technical tabletop with a warm neutral background and subtle ALUX-like material language. Show one restrained request origin feeding four separate glass-and-metal rails, each carrying a different search capsule. The lanes never intersect and the searches finish independently; do not show message exchange, waiting, fusion, or four sudden success badges. High-end editorial 2.5D technical illustration, realistic materials, precise and polished, suitable for a serious technology website. Wide 16:9, strong left-to-right direction, lanes fully visible, enough clear negative space around edges for responsive cropping. Crisp, energetic but calm lighting with warm highlights and understated teal and amber signals. Palette: ivory, graphite, muted teal, amber, and a little soft violet; avoid monochrome purple or blue.

## `concurrent.webp`

Use case: stylized-concept. Asset type: website section illustration for mode state: Concurrent.

Create a premium technical illustration that explains message-driven concurrency through a hotel booking. Two different transactions interact through a visible channel: the booking leaves a room-availability condition, and a hotel service later sends the matching availability message. The match creates a new continuation that carries a room key forward, while unrelated work keeps moving. Do not depict concurrency as one capsule merely stopping and restarting at the same physical point. The scene is a clean warm technical tabletop with two clear rails, one central channel match, and no labels. High-end editorial 2.5D technical illustration, realistic glass, metal, and ceramic materials, sophisticated product-site visual. Wide 16:9, readable causal sequence, all parts fully visible. Calm, precise, and premium. Palette: ivory, graphite, muted teal, amber, and restrained violet.

## `composable.webp`

Use case: stylized-concept. Asset type: website section illustration for mode state: Composable.

Create a premium technical illustration that explains cross-system atomic composition through one real booking. A hotel reservation system holds a room while a payment system authorizes the charge. Their two distinct state transitions enter one transparent atomic boundary; a single booking confirmation may leave only after both sides are ready. If either side fails, no confirmation leaves. Do not show four transactions being fused into one larger capsule. The scene is a clean warm technical studio with two clearly different source systems, an elegant central lock or settlement chamber, and one restrained confirmation dock. High-end editorial 2.5D technical illustration, realistic glass, metal, and ceramic materials, refined product-site aesthetic, no diagrammatic text. Wide 16:9, strong visual clarity, objects fully visible, no labels. Composed, satisfying, and understandable to a nontechnical audience. Palette: ivory, charcoal, muted teal, amber, restrained violet, and one small green confirmation light.
